## Run this project locally in VS Code

This is a TanStack Start + Vite + React app with Lovable Cloud (Supabase) for the contact form. Below are the exact steps to get it running on your laptop.

### 1. Install prerequisites

- **Node.js 20+** — download from https://nodejs.org (LTS).
- **Bun** (this project uses Bun as its package manager — `bun.lockb` is committed):
  - macOS / Linux: `curl -fsSL https://bun.sh/install | bash`
  - Windows: `powershell -c "irm bun.sh/install.ps1 | iex"`
  - (If you'd rather use npm, you can — just delete `bun.lockb` first and use `npm install`.)
- **VS Code** — https://code.visualstudio.com
- Recommended VS Code extensions: ESLint, Prettier, Tailwind CSS IntelliSense.

### 2. Get the code onto your laptop

In Lovable, click the **GitHub** button (top right) → **Connect to GitHub** → push the project to a new repo. Then on your laptop:

```bash
git clone https://github.com/<your-username>/<your-repo>.git
cd <your-repo>
code .
```

(Alternatively: GitHub → "Code" → "Download ZIP" → extract → open the folder in VS Code.)

### 4. Install dependencies and run

In the VS Code integrated terminal (`` Ctrl+` ``):

```bash
bun install
bun run dev
```

Open the URL it prints (usually **http://localhost:5173**) in your browser.

Other useful commands:
- `bun run build` — production build
- `bun run preview` — preview the production build
- `bun run lint` — run ESLint

### 5. Keeping in sync with Lovable

Once GitHub is connected, edits made in Lovable auto-push to the repo, and edits you push from your laptop auto-sync into Lovable. Pull before you start working: `git pull`.

### Technical notes

- Stack: TanStack Start v1 (file-based routing in `src/routes/`), Vite 7, React 19, Tailwind v4, shadcn/ui, Supabase JS client.
- Package manager: Bun (lockfile `bun.lockb`). npm/pnpm work too if you remove the lockfile.
- The dev server runs in Node — no Cloudflare Worker emulation locally.
- `src/integrations/supabase/client.ts` and `types.ts` are auto-generated by Lovable; don't edit them by hand.
- If `bun run dev` fails with a missing-env error, double-check the `.env` file is in the project root (same folder as `package.json`), not inside `src/`.
